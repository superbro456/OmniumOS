# Omnium OS (Aurora)

**Модульная гибридная операционная система** с адаптивным ядром **FlexKernel**, зональным управлением ресурсами и глубоко интегрированной многоуровневой безопасностью.

![Архитектура Omnium OS](docs/architecture.svg)

---

## 1. Философия и цели

Omnium OS проектируется как **платформенная операционная система**, а не просто «ядро + user space».

Система ориентирована на предсказуемость, масштабируемость и безопасность на уровне архитектуры.

### Ключевые цели:

- Детерминированное поведение ядра  
- Масштабируемость от **MONO** до **DISTRIBUTED**  
- Строгая изоляция компонентов без потери производительности  
- Встроенная безопасность как базовое свойство системы  
- Интеграция с кастомной архитектурой **OmniumX**  

---

## 2. Архитектура ядра: FlexKernel

**FlexKernel** — гибридное ядро, объединяющее:

- микроядерный подход (изоляция, IPC, сервисная модель)
- монолитный подход (производительность, низкая латентность)

Ядро разделено на независимые **Core-модули**, каждый из которых выполняет строго определённую функцию.

---

### 2.1 Core-модули ядра

#### Central Core
Главный координатор системы.

Функции:
- управление режимами (**MONO / DISTRIBUTED / EMERGENCY**)
- маршрутизация системных событий
- синхронизация глобальных состояний

---

#### Process Core
Подсистема управления процессами и CPU-временем.

Функции:
- планирование задач
- управление приоритетами
- распределение CPU-бюджетов
- переключение контекста
- управление состояниями процессов

Характеристики:
- per-CPU runqueue (без глобальных очередей)
- строгая иерархия приоритетов
- бюджетно-кредитная модель
- поддержка добровольной отдачи CPU (`wants_cpu`)

Тип планировщика:
- Per-CPU планировщик с жесткими приоритетами, бюджетированием и перераспределением кредитов


---

#### Memory Core
Централизованный слой управления физической памятью.

Ключевые свойства:
- policy-based allocation
- зональная модель памяти
- детерминированная аллокация
- отсутствие глобальных блокировок

Зоны памяти:
- **Kernel Zone** — ядро и критические структуры  
- **Application Zone** — пользовательские процессы  
- **Service Zone** — системные сервисы  

---

#### Security Core
Центр принятия решений безопасности.

Функции:
- контроль доступа
- аудит
- управление карантином
- реакция на угрозы в реальном времени

---

#### AI Core
Интеллектуальный аналитический модуль.

Функции:
- анализ поведения системы
- прогнозирование нагрузки
- обнаружение аномалий
- помощь Security Core

---

#### Metrics Core
Система телеметрии.

Функции:
- сбор метрик
- передача данных в AI Core и Optimize Core

---

#### Optimize Core
Адаптивная оптимизация системы.

Функции:
- динамическая балансировка нагрузки
- NUMA-aware распределение ресурсов
- оптимизация производительности в реальном времени

---

## 3. Режимы работы системы

- **MONO**  
  Минимальная конфигурация, максимальная предсказуемость  

- **DISTRIBUTED**  
  Полная параллельная обработка и высокая отказоустойчивость  

- **EMERGENCY**  
  Приоритет производительности над изоляцией  

---

## 4. Plants — изолированные домены

Система разделена на изолированные сервисные домены (**Plants**):

| Plant     | Назначение                          |
|-----------|-------------------------------------|
| Plant 1   | Сеть, I/O, драйверы                 |
| Plant 2   | Пользовательские приложения, GUI    |
| Plant 3   | Системные сервисы                   |
| Plant 4   | Песочница / карантин                |

---

## 5. FlexIPC

Безопасная межмодульная коммуникационная шина.

Особенности:
- mTLS
- шифрование
- zero-trust модель
- строгая изоляция взаимодействий

---

## 6. OmniumFS (OFS3)

Современная файловая система.

Особенности:
- объектная модель хранения
- версионирование
- проверка целостности
- высокая устойчивость к сбоям

---

## 7. Многоуровневая система защиты

Security Core управляет **6 уровнями защиты**:

1. **Aethis Layer** — сетевой периметр  
2. **Phantom Layer** — обман и ловушки  
3. **Voltbreaker Layer** — активное вмешательство  
4. **Ironveil Layer** — защита ядра и памяти  
5. **Shadow Bastion** — скрытые пространства  
6. **Core Sanctum** — финальная защита ядра и OFS3  

---

## 8. Архитектурная классификация

- Hybrid Kernel Architecture  
- Zone-based Memory Management  
- Policy-driven Resource Allocation  
- Integrated Multi-layer Security OS  
- Platform-oriented Operating System  

---

## 9. Общая концепция

**Omnium OS** — это система, где:

- планирование
- память
- безопасность
- хранение данных

представляют собой **единую архитектуру**, а не набор разрозненных подсистем.
